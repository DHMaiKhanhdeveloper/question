QUESTION 31 · TEST AUTOMATION

What is performance testing, and have you done load testing?

Performance testing là gì, và bạn đã làm load testing chưa?

Sample Answer (English)

Performance testing is the process of checking how a system behaves under a given workload, focusing on metrics like response time, throughput, and resource usage instead of functional correctness. Load testing is one type of performance testing where I simulate expected or peak numbers of concurrent users to see if the system meets its performance requirements. I have also done stress testing, pushing traffic beyond normal limits to find the breaking point, and spike testing, to see how the system reacts to a sudden surge. In my projects, I used k6 and JMeter to write load test scripts, ramp virtual users gradually, and collect metrics such as response time percentiles, error rate, and requests per second. Then I analyze the results together with server monitoring dashboards to identify bottlenecks, like slow database queries or CPU saturation, and report them to the development team.

💻 Ví dụ minh họa (k6 load test script)

// load-test.js — mô phỏng 50 virtual users trong 1 phút
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '30s', target: 50 },  // ramp-up lên 50 VUs
    { duration: '1m',  target: 50 },  // giữ tải ổn định
    { duration: '15s', target: 0 },   // ramp-down
  ],
  thresholds: {
    http_req_duration: ['p(95)<500'], // 95% request < 500ms
    http_req_failed: ['rate<0.01'],   // error rate < 1%
  },
};

export default function () {
  const res = http.get('https://api.example.com/products');
  check(res, { 'status is 200': (r) => r.status === 200 });
  sleep(1);
}
