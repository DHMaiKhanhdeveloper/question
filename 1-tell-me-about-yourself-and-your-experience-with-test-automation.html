QUESTION 1 · TEST AUTOMATION

Tell me about yourself and your experience with test automation.

Hãy giới thiệu bản thân và kinh nghiệm automation testing của bạn.

Sample Answer (English)

I'm a QA engineer with experience in both manual and automated testing. For automation, I mainly work with Playwright and Selenium to test web applications. I write end-to-end tests, build reusable frameworks using the Page Object Model, and integrate the tests into CI/CD pipelines so they run automatically on every build. My focus is on writing stable, maintainable tests that reduce flakiness and give the team fast feedback on quality.

🇻🇳 Bản dịch tiếng Việt

Tôi là một kỹ sư QA, có kinh nghiệm cả về kiểm thử thủ công lẫn kiểm thử tự động. Về phần tự động hóa, tôi chủ yếu làm việc với Playwright và Selenium để kiểm thử các ứng dụng web. Tôi viết các bài kiểm thử đầu-cuối, xây dựng những bộ khung tái sử dụng được theo mô hình Page Object Model, và tích hợp các bài kiểm thử vào quy trình CI/CD để chúng tự động chạy mỗi khi có bản dựng mới. Tôi tập trung vào việc viết những bài kiểm thử ổn định, dễ bảo trì, giúp giảm tình trạng chập chờn và mang lại phản hồi nhanh về chất lượng cho cả nhóm.

Gợi ý nội dung cần nêu

  • Số năm kinh nghiệm & loại ứng dụng (web / mobile / API)
  • Công cụ: Playwright, Selenium
  • Ngôn ngữ lập trình: JavaScript/TypeScript, Python hoặc Java
  • Mô hình framework: Page Object Model (POM)
  • CI/CD: Jenkins, GitHub Actions, GitLab CI...
  • Kết quả đạt được: giảm thời gian regression, tăng độ ổn định

🌟 Ví dụ thực tế để kể thêm

"For example, in my last project — an e-commerce website — I built an end-to-end test suite with Playwright that covered the whole checkout flow. It ran in our GitHub Actions pipeline on every pull request, and it cut our regression testing from about two days of manual work down to roughly 15 minutes."

👉 Ví dụ (dịch): "Ví dụ, ở dự án gần nhất — một trang thương mại điện tử — tôi xây dựng bộ test đầu-cuối bằng Playwright bao phủ toàn bộ luồng thanh toán. Nó chạy trong pipeline GitHub Actions ở mỗi pull request, giúp giảm thời gian kiểm thử hồi quy từ khoảng 2 ngày làm thủ công xuống còn ~15 phút."

💻 Ví dụ code: một test E2E đơn giản (Playwright)

// checkout.spec.js — kiểm thử luồng thanh toán
import { test, expect } from '@playwright/test';

test('user can checkout successfully', async ({ page }) => {
  await page.goto('https://shop.example.com');
  await page.getByRole('button', { name: 'Add to cart' }).click();
  await page.getByRole('link', { name: 'Checkout' }).click();
  await page.getByLabel('Card number').fill('4242 4242 4242 4242');
  await page.getByRole('button', { name: 'Pay now' }).click();
  // assertion tự chờ đến khi thấy thông báo
  await expect(page.getByText('Order confirmed')).toBeVisible();
});
