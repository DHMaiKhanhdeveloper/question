QUESTION 5 · TEST AUTOMATION

What is CI/CD, and how do you integrate automated tests into a pipeline?

CI/CD là gì, và bạn tích hợp automated test vào pipeline như thế nào?

Sample Answer (English)

CI/CD stands for Continuous Integration and Continuous Deployment. Continuous Integration means every code change is automatically built and tested as soon as it's pushed, so bugs are caught early instead of piling up. Continuous Deployment means that if those tests pass, the code can move automatically toward staging or production. To integrate automated tests, I add a test stage in the pipeline configuration, for example a GitHub Actions workflow, that installs dependencies, then runs my Playwright or Selenium test suite on every pull request. I usually split tests into fast smoke tests that run first, and a fuller regression suite that runs in parallel to save time. If any test fails, the pipeline fails fast and blocks the merge, and the test report is uploaded as an artifact so the team can quickly see what broke.

🇻🇳 Bản dịch tiếng Việt

CI/CD là viết tắt của Continuous Integration và Continuous Deployment. Continuous Integration nghĩa là mỗi thay đổi code đều được tự động build và test ngay khi được push lên, nhờ vậy lỗi được phát hiện sớm thay vì dồn lại. Continuous Deployment nghĩa là nếu các test đó pass, code có thể tự động được đưa tới môi trường staging hoặc production. Để tích hợp automated test, tôi thêm một test stage vào cấu hình pipeline, ví dụ một GitHub Actions workflow, để cài đặt các dependency rồi chạy bộ test suite Playwright hoặc Selenium của tôi trên mỗi pull request. Tôi thường chia test thành các smoke test nhanh chạy trước, và một regression suite đầy đủ hơn chạy song song (parallel) để tiết kiệm thời gian. Nếu có test nào fail, pipeline sẽ fail fast và chặn việc merge, đồng thời test report được upload dưới dạng artifact để cả team nhanh chóng biết chỗ nào bị lỗi.

💻 Ví dụ minh họa (GitHub Actions)

# .github/workflows/tests.yml — chạy Playwright test trên mỗi push/PR
name: Playwright Tests
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      # 1. Lấy code
      - uses: actions/checkout@v4

      # 2. Cài dependency
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm ci
      - run: npx playwright install --with-deps

      # 3. Smoke test nhanh trước — fail fast
      - name: Smoke tests
        run: npx playwright test --grep @smoke

      # 4. Regression suite chạy song song (sharding)
      - name: Regression tests
        run: npx playwright test --shard=1/4

      # 5. Upload test report làm artifact dù pass hay fail
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: playwright-report
          path: playwright-report/
