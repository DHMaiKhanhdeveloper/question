QUESTION 15 · TEST AUTOMATION

What is the difference between unit, integration, and end-to-end tests?

Sự khác biệt giữa unit test, integration test và end-to-end test là gì?

Sample Answer (English)

These three types of tests differ mainly in scope and speed. A unit test checks a single function or class in isolation, usually with all external dependencies mocked, so it runs in milliseconds and pinpoints exactly where a bug is. An integration test verifies that two or more real components, like a service and a database, work correctly together, so it's slower and catches issues that mocks would hide, such as a wrong SQL query. An end-to-end test drives the whole application through the real user interface, simulating an actual user flow like login and checkout, using a tool such as Playwright. It's the slowest and most brittle, but it gives the highest confidence that the system works as a whole. I usually follow the testing pyramid: many unit tests, fewer integration tests, and only a small number of end-to-end tests for critical paths.

🇻🇳 Bản dịch tiếng Việt

Ba loại test này khác nhau chủ yếu ở phạm vi (scope) và tốc độ. Một unit test kiểm tra một hàm hoặc một class duy nhất một cách độc lập, thường với tất cả các phụ thuộc bên ngoài được giả lập (mock), nên nó chạy trong vài mili-giây và chỉ ra chính xác lỗi nằm ở đâu. Một integration test kiểm tra xem hai hay nhiều thành phần thật, ví dụ một service và một database, có hoạt động đúng cùng nhau hay không, nên nó chậm hơn và bắt được những lỗi mà mock sẽ che giấu, chẳng hạn một câu truy vấn SQL sai. Một end-to-end test vận hành toàn bộ ứng dụng thông qua giao diện người dùng thật, mô phỏng một luồng người dùng thực tế như đăng nhập và thanh toán, bằng một công cụ như Playwright. Đây là loại chậm nhất và dễ vỡ nhất, nhưng nó mang lại độ tin cậy cao nhất rằng toàn bộ hệ thống hoạt động đúng. Tôi thường tuân theo mô hình kim tự tháp kiểm thử: nhiều unit test, ít integration test hơn, và chỉ một số lượng nhỏ end-to-end test cho các luồng quan trọng nhất.

💻 Ví dụ minh họa (Jest & Playwright)

// 1) UNIT TEST — mock database, chỉ test logic thuần
function calcDiscount(price, isVip) {
  return isVip ? price * 0.8 : price;
}
test('applies 20% discount for VIP', () => {
  expect(calcDiscount(100, true)).toBe(80);
});

// 2) INTEGRATION TEST — dùng service thật + DB test thật
test('orderService saves order to real DB', async () => {
  const order = await orderService.create({ userId: 1, price: 100 });
  const saved = await db.orders.findById(order.id);
  expect(saved.price).toBe(100);
});

// 3) END-TO-END TEST — Playwright chạy trên UI thật
test('user can checkout an order', async ({ page }) => {
  await page.goto('/login');
  await page.getByTestId('username').fill('admin');
  await page.getByRole('button', { name: 'Checkout' }).click();
  await expect(page.getByText('Order confirmed')).toBeVisible();
});
