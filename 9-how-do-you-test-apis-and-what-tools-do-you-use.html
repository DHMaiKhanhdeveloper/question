QUESTION 9 · TEST AUTOMATION

How do you test APIs, and what tools do you use?

Bạn test API như thế nào, và bạn dùng công cụ nào?

Sample Answer (English)

When I test APIs, I usually work at two levels: exploratory testing with Postman during development, and automated regression tests in the framework, using REST Assured for Java projects or Playwright's APIRequestContext for JavaScript projects. For each endpoint, I send requests with different inputs, including valid data, invalid data, and edge cases, then verify the status code, the response body against a JSON schema, and important headers like content type. I also check response time to catch performance regressions, and I make sure error handling returns the correct status code and a clear error message for bad requests, unauthorized access, or missing resources. I usually chain these API tests into the CI pipeline so they run automatically before every deployment, which catches backend issues early, before they ever reach the UI.

🇻🇳 Bản dịch tiếng Việt

Khi test API, tôi thường làm việc ở hai mức độ: test thăm dò (exploratory testing) bằng Postman trong lúc phát triển, và các bài test hồi quy tự động trong framework, dùng REST Assured cho dự án Java hoặc APIRequestContext của Playwright cho dự án JavaScript. Với mỗi endpoint, tôi gửi request với nhiều loại dữ liệu đầu vào khác nhau, bao gồm dữ liệu hợp lệ, dữ liệu không hợp lệ, và trường hợp biên (edge case), sau đó kiểm tra mã trạng thái (status code), phần thân phản hồi (response body) theo một JSON schema, và các header quan trọng như content type. Tôi cũng kiểm tra thời gian phản hồi để phát hiện suy giảm hiệu năng, và đảm bảo việc xử lý lỗi trả về đúng mã trạng thái cùng thông báo lỗi rõ ràng cho các request sai, truy cập trái phép, hoặc tài nguyên không tồn tại. Tôi thường đưa các test API này vào CI pipeline để chúng chạy tự động trước mỗi lần triển khai, giúp phát hiện sớm các vấn đề ở backend trước khi chúng ảnh hưởng tới giao diện người dùng.

💻 Ví dụ minh họa (Playwright APIRequestContext)

// api.spec.js — test API dùng Playwright APIRequestContext
const { test, expect } = require('@playwright/test');

test('GET /users/:id returns valid user schema', async ({ request }) => {
  const res = await request.get('/api/users/42');

  // 1. Kiểm tra status code
  expect(res.status()).toBe(200);

  // 2. Kiểm tra response body / schema
  const body = await res.json();
  expect(body).toHaveProperty('id', 42);
  expect(body).toHaveProperty('email');
  expect(typeof body.email).toBe('string');

  // 3. Kiểm tra header
  expect(res.headers()['content-type']).toContain('application/json');
});

test('POST /users with missing field returns 400', async ({ request }) => {
  const res = await request.post('/api/users', {
    data: { name: 'Duy' } // thiếu field "email" bắt buộc
  });
  expect(res.status()).toBe(400);
  const err = await res.json();
  expect(err.message).toContain('email');
});
