QUESTION 27 · TEST AUTOMATION

How do you test mobile applications or responsive web apps?

Bạn test ứng dụng mobile hoặc web responsive như thế nào?

Sample Answer (English)

I split this into two areas. For a responsive web app, I use Playwright's viewport emulation and device descriptors, like devices['iPhone 13'], to simulate different screen sizes, pixel ratios, and touch input without needing a real phone. I check that layout adapts correctly across the common responsive breakpoints, that touch gestures such as tap, swipe, and pinch-zoom work, and that content isn't cut off or overlapping. For a native or hybrid mobile app, viewport emulation isn't enough because I need to interact with real OS-level components, so I use Appium, which drives the app through the actual accessibility tree on a real device or emulator, covering things like permission dialogs, push notifications, and orientation changes. To scale coverage across many device and OS combinations, I run these suites on cloud device farms like BrowserStack or Sauce Labs instead of maintaining physical devices myself.

🇻🇳 Bản dịch tiếng Việt

Tôi chia việc này thành hai hướng. Với web responsive, tôi dùng tính năng viewport emulation và device descriptors của Playwright, ví dụ devices['iPhone 13'], để mô phỏng các kích thước màn hình, tỉ lệ pixel và thao tác chạm khác nhau mà không cần điện thoại thật. Tôi kiểm tra bố cục có thích ứng đúng qua các breakpoint responsive phổ biến không, các cử chỉ chạm (touch gestures) như tap, swipe, pinch-zoom có hoạt động không, và nội dung có bị cắt hoặc chồng lấp không. Với ứng dụng mobile native hoặc hybrid, viewport emulation là chưa đủ vì tôi cần tương tác với các thành phần cấp hệ điều hành thật, nên tôi dùng Appium, công cụ điều khiển ứng dụng thông qua accessibility tree thật trên thiết bị thật hoặc emulator, bao gồm cả hộp thoại xin quyền, thông báo đẩy và thay đổi hướng màn hình. Để mở rộng độ phủ qua nhiều tổ hợp thiết bị và hệ điều hành, tôi chạy các bộ test này trên các cloud device farm như BrowserStack hoặc Sauce Labs thay vì tự duy trì thiết bị vật lý.

💻 Ví dụ minh họa (Playwright device emulation)

// playwright.config.js — emulate a real device profile
import { devices } from '@playwright/test';

export default {
  projects: [
    {
      name: 'Mobile Chrome',
      use: { ...devices['Pixel 7'] },
    },
    {
      name: 'Mobile Safari',
      use: { ...devices['iPhone 13'] },
    },
  ],
};

// test file — assert layout + touch gesture at a breakpoint
test('nav collapses into hamburger on small viewport', async ({ page }) => {
  await page.setViewportSize({ width: 375, height: 812 });
  await page.goto('/');
  await expect(page.getByRole('button', { name: 'Menu' })).toBeVisible();
  await page.getByTestId('carousel').tap();
});
