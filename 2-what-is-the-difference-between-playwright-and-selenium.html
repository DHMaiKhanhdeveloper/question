QUESTION 2 · TEST AUTOMATION

What is the difference between Playwright and Selenium, and when would you use each?

Sự khác nhau giữa Playwright và Selenium là gì, và khi nào bạn dùng công cụ nào?

Sample Answer (English)

Both Playwright and Selenium are popular tools for browser automation, but they work a bit differently. Selenium has been around much longer, it supports many programming languages and browsers, and it has a huge community, so it's a great choice for legacy or cross-browser projects. Playwright is newer; it has built-in auto-waiting, runs faster, and supports modern features like network interception out of the box, which makes the tests less flaky and easier to write. For a brand-new project I usually prefer Playwright, but if a team already has a stable framework or needs very broad browser support, I'm completely comfortable working with Selenium too.

🇻🇳 Bản dịch tiếng Việt

Cả Playwright và Selenium đều là những công cụ phổ biến để tự động hóa trình duyệt, nhưng cách hoạt động hơi khác nhau. Selenium xuất hiện lâu hơn nhiều, hỗ trợ nhiều ngôn ngữ lập trình và nhiều trình duyệt, lại có cộng đồng rất lớn, nên rất phù hợp cho các dự án cũ (legacy) hoặc cần kiểm thử đa trình duyệt. Playwright thì mới hơn; nó có cơ chế tự động chờ sẵn có, chạy nhanh hơn, và hỗ trợ sẵn các tính năng hiện đại như chặn/bắt request mạng (out of the box), giúp các bài kiểm thử ít chập chờn hơn và dễ viết hơn. Với một dự án hoàn toàn mới, tôi thường ưu tiên Playwright, nhưng nếu nhóm đã có sẵn một bộ khung ổn định bằng Selenium hoặc cần hỗ trợ rất nhiều trình duyệt, tôi cũng hoàn toàn thoải mái khi làm việc với Selenium.

💻 Ví dụ code: cùng một thao tác, 2 công cụ

Playwright — tự động chờ, ngắn gọn:

// Playwright (JS) — auto-wait sẵn, không cần khai báo wait
await page.getByRole('button', { name: 'Login' }).click();
await expect(page.getByText('Welcome')).toBeVisible();

Selenium — phải tự khai báo wait:

# Selenium (Python) — cần WebDriverWait để tránh flaky
wait = WebDriverWait(driver, 10)
btn = wait.until(EC.element_to_be_clickable((By.ID, "login")))
btn.click()
msg = wait.until(EC.visibility_of_element_located((By.ID, "welcome")))
assert msg.is_displayed()

👉 Cùng một việc "bấm nút rồi kiểm tra thông báo", nhưng Playwright ngắn hơn nhờ auto-waiting, còn Selenium phải tự viết wait — đây chính là lý do Playwright ít flaky hơn.
