QUESTION 6 · TEST AUTOMATION

How do you handle test data management in automation?

Bạn quản lý test data trong automation như thế nào?

Sample Answer (English)

I try to keep test data independent from the UI and from other tests, so each test can run in isolation and in any order. Whenever possible, I seed data directly through API calls or database calls before the test starts, instead of creating it manually through the UI, because that's much faster and more reliable. I also use factory functions and fixtures to generate realistic data, often combined with a library like Faker for random values, so tests don't collide when running in parallel. For cleanup, I make sure every test tears down the data it created, either through an API call or a database rollback, so the environment stays clean. For different environments, I keep environment-specific configuration and datasets separate, so the same suite can run safely against staging or a local environment.

🇻🇳 Bản dịch tiếng Việt

Tôi cố gắng giữ cho test data độc lập với giao diện và với các test khác, để mỗi test có thể chạy độc lập (isolation) và theo bất kỳ thứ tự nào. Bất cứ khi nào có thể, tôi tạo sẵn (seed) dữ liệu trực tiếp thông qua các lệnh gọi API hoặc lệnh gọi cơ sở dữ liệu trước khi test bắt đầu, thay vì tạo thủ công qua giao diện, vì cách đó nhanh và đáng tin cậy hơn nhiều. Tôi cũng dùng các hàm factory và fixture để sinh ra dữ liệu thực tế, thường kết hợp với thư viện như Faker để tạo giá trị ngẫu nhiên, để các test không bị xung đột khi chạy song song. Về dọn dẹp (cleanup), tôi đảm bảo mỗi test đều xóa bỏ dữ liệu mà nó đã tạo ra, thông qua lệnh gọi API hoặc rollback cơ sở dữ liệu, để môi trường luôn sạch sẽ. Đối với các môi trường khác nhau, tôi giữ cấu hình theo từng môi trường và bộ dữ liệu riêng, để cùng một bộ test có thể chạy an toàn trên staging hoặc môi trường local.

💻 Ví dụ minh họa (Playwright)

// factories.js — factory function sinh dữ liệu thực tế bằng Faker
import { faker } from '@faker-js/faker';

export function buildUser(overrides = {}) {
  return {
    email: faker.internet.email(),
    password: 'P@ssw0rd123',
    fullName: faker.person.fullName(),
    ...overrides,
  };
}

// test.spec.js — seed qua API trước khi test, cleanup sau khi test
import { test, expect } from '@playwright/test';
import { buildUser } from './factories';

test('user can update profile', async ({ page, request }) => {
  const user = buildUser();

  // Seed dữ liệu trực tiếp qua API, không đi qua UI
  const created = await request.post('/api/users', { data: user });
  const { id } = await created.json();

  await page.goto(`/profile/${id}`);
  await expect(page.getByText(user.fullName)).toBeVisible();

  // Teardown — xóa data đã tạo để không ảnh hưởng test khác
  await request.delete(`/api/users/${id}`);
});
