QUESTION 28 · TEST AUTOMATION

What is a test plan, and what does it include?

Test plan là gì, và nó bao gồm những gì?

Sample Answer (English)

A test plan is a formal document that describes the test strategy and approach for testing a software product. It typically starts by defining the scope, meaning which features or modules are in scope and which are out of scope, along with the objectives we want to achieve. It also outlines the test strategy, describing the types of testing we'll perform, like functional, regression, and performance testing, and the tools or environments we'll use. A good test plan defines entry criteria and exit criteria, so everyone knows when testing can start and when it's considered complete. It lists the resources and timeline, including team roles and the schedule. Finally, it includes a risk assessment, identifying potential risks and mitigation plans. Overall, the test plan aligns the whole team on what, how, and when we're testing.

🇻🇳 Bản dịch tiếng Việt

Test plan là một tài liệu chính thức mô tả chiến lược kiểm thử (test strategy) và cách tiếp cận để kiểm thử một sản phẩm phần mềm. Nó thường bắt đầu bằng việc xác định phạm vi (scope), nghĩa là những tính năng hoặc module nào nằm trong phạm vi và những cái nào không, cùng với các mục tiêu (objectives) mà chúng ta muốn đạt được. Nó cũng phác thảo chiến lược kiểm thử, mô tả các loại kiểm thử sẽ thực hiện, như kiểm thử chức năng, kiểm thử hồi quy và kiểm thử hiệu năng, cùng các công cụ hoặc môi trường sẽ sử dụng. Một test plan tốt xác định tiêu chí bắt đầu (entry criteria) và tiêu chí kết thúc (exit criteria), để mọi người biết khi nào có thể bắt đầu kiểm thử và khi nào được coi là hoàn thành. Nó liệt kê nguồn lực (resources) và thời gian biểu (timeline), bao gồm vai trò của từng thành viên và lịch trình. Cuối cùng, nó bao gồm phần đánh giá rủi ro (risk assessment), xác định các rủi ro (risks) tiềm ẩn và kế hoạch giảm thiểu (mitigation plans). Nhìn chung, test plan giúp cả nhóm thống nhất (align) về việc kiểm thử cái gì, như thế nào, và khi nào.

💻 Cấu trúc Test Plan mẫu

// Cấu trúc điển hình của một Test Plan (theo chuẩn IEEE 829)
1. Test Plan ID          // mã định danh tài liệu
2. Introduction / Objectives  // mục tiêu kiểm thử
3. Scope                 // in-scope / out-of-scope
4. Test Strategy / Approach   // loại test, kỹ thuật
5. Test Items & Features     // module/tính năng cần test
6. Entry Criteria        // điều kiện bắt đầu test
7. Exit Criteria         // điều kiện kết thúc test
8. Test Environment      // môi trường, thiết bị, data
9. Resources & Roles     // nhân sự, trách nhiệm
10. Schedule / Timeline  // lịch trình các giai đoạn test
11. Deliverables         // test case, báo cáo, bug log
12. Risk Assessment      // rủi ro & kế hoạch giảm thiểu
13. Approval             // người phê duyệt tài liệu
