QUESTION 26 · TEST AUTOMATION

How do you handle file uploads and downloads in automated tests?

Bạn xử lý upload và download file trong automated test như thế nào?

Sample Answer (English)

For file uploads, I use Playwright's setInputFiles method on the hidden file input element, passing an absolute path to a fixture file, which avoids dealing with the native OS dialog completely. For downloads, I listen for the download event using page.waitForEvent('download') before triggering the action that starts the download, then I call saveAs to store the file, usually in a temp directory created for that test run. After that, I verify the downloaded file by checking its suggestedFilename, its file size, and sometimes reading its content to confirm it matches the expected data. I always clean up temp directories after the test to avoid leftover files. This approach keeps upload and download tests fast and reliable, independent of the OS file picker.

🇻🇳 Bản dịch tiếng Việt

Đối với upload file, tôi dùng phương thức setInputFiles của Playwright trên phần tử file input bị ẩn, truyền vào đường dẫn tuyệt đối tới một file fixture, nhờ đó tránh hoàn toàn việc phải xử lý hộp thoại gốc của hệ điều hành. Đối với download, tôi lắng nghe sự kiện download bằng page.waitForEvent('download') trước khi kích hoạt hành động bắt đầu tải file, sau đó gọi saveAs để lưu file, thường vào một thư mục tạm được tạo riêng cho lần chạy test đó. Sau đó, tôi xác minh file đã tải xuống bằng cách kiểm tra suggestedFilename, kích thước file, và đôi khi đọc nội dung file để xác nhận nó khớp với dữ liệu mong đợi. Tôi luôn dọn dẹp các thư mục tạm sau khi test xong để tránh còn sót lại file thừa. Cách tiếp cận này giúp test upload và download nhanh và đáng tin cậy, không phụ thuộc vào hộp thoại chọn file của hệ điều hành.

💻 Ví dụ minh họa (Playwright)

// 1) UPLOAD — dùng setInputFiles, không cần dialog OS
await page.setInputFiles('#avatar-upload', './fixtures/avatar.png');

// Upload nhiều file cùng lúc
await page.setInputFiles('#docs-upload', [
  './fixtures/file1.pdf',
  './fixtures/file2.pdf'
]);

// 2) DOWNLOAD — chờ sự kiện download trước khi click
const [download] = await Promise.all([
  page.waitForEvent('download'),
  page.getByRole('button', { name: 'Export CSV' }).click(),
]);

// Lưu file vào thư mục tạm và kiểm tra tên / nội dung
const path = './tmp/' + download.suggestedFilename();
await download.saveAs(path);
expect(download.suggestedFilename()).toBe('report.csv');
