QUESTION 10 · TEST AUTOMATION

What is cross-browser testing, and how do you implement it?

Cross-browser testing là gì, và bạn triển khai nó như thế nào?

Sample Answer (English)

Cross-browser testing is the practice of verifying that a web application behaves and renders correctly across different browsers, browser engines, and devices, since each engine can handle CSS, JavaScript, and layout slightly differently. In my automation work, I rely on Playwright because it ships with three rendering engines out of the box: Chromium for Chrome and Edge, Firefox's Gecko engine, and WebKit, which mirrors Safari. I define separate projects in the playwright.config.js file, each pointing to a different browser and device profile, including mobile viewport emulation, and then run the same test suite against all of them in parallel. For broader real-device coverage, especially older browser versions or actual Safari on macOS, I integrate cloud grids like BrowserStack or Sauce Labs into the CI pipeline.

🇻🇳 Bản dịch tiếng Việt

Cross-browser testing là quá trình kiểm tra xem một ứng dụng web có hoạt động và hiển thị đúng trên các trình duyệt, engine trình duyệt và các thiết bị khác nhau hay không, vì mỗi engine có thể xử lý CSS, JavaScript và bố cục (layout) theo cách hơi khác nhau. Trong công việc tự động hóa, tôi dùng Playwright vì nó tích hợp sẵn ba engine kết xuất: Chromium cho Chrome và Edge, engine Gecko của Firefox, và WebKit, tương ứng với Safari. Tôi định nghĩa các project riêng biệt trong file playwright.config.js, mỗi project trỏ tới một trình duyệt và cấu hình thiết bị khác nhau, bao gồm cả giả lập màn hình di động, rồi chạy cùng một bộ test song song trên tất cả các trình duyệt đó. Để mở rộng phạm vi kiểm thử trên thiết bị thật, đặc biệt là các phiên bản trình duyệt cũ hoặc Safari thật trên macOS, tôi tích hợp các cloud grid như BrowserStack hoặc Sauce Labs vào CI pipeline.

💻 Ví dụ minh họa (Playwright)

// playwright.config.js — chạy song song trên nhiều engine + mobile
const { defineConfig, devices } = require('@playwright/test');

module.exports = defineConfig({
  testDir: './tests',
  fullyParallel: true,
  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
    },
    {
      name: 'firefox',
      use: { ...devices['Desktop Firefox'] },
    },
    {
      name: 'webkit',
      use: { ...devices['Desktop Safari'] },
    },
    {
      name: 'mobile-chrome',
      use: { ...devices['Pixel 7'] },
    },
    {
      name: 'mobile-safari',
      use: { ...devices['iPhone 14'] },
    },
  ],
});

🌐 Trình duyệt / Engine cần bao phủ

Trình duyệt / EngineLưu ý khi test
Chromium (Chrome, Edge)Thị phần lớn nhất; hầu hết bug xuất hiện đầu tiên ở đây được coi là "chuẩn"
Firefox (Gecko)Xử lý CSS grid/flexbox và font-rendering đôi khi khác Chromium
WebKit / SafariThường gây bug nhiều nhất: khác biệt về Web API, date picker, video, IndexedDB
Mobile viewport emulationKiểm tra responsive layout, touch events, kích thước màn hình nhỏ
Trình duyệt phiên bản cũPlaywright không cover được — cần cloud grid (BrowserStack, Sauce Labs)
