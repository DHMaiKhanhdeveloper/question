QUESTION 19 · TEST AUTOMATION

How do you report and track bugs found during testing?

Bạn báo cáo và theo dõi bug phát hiện được trong quá trình test như thế nào?

Sample Answer (English)

Whenever I find a defect, I log it in a tracking tool like Jira with a clear title and detailed reproduction steps, so anyone can reproduce the issue without asking me questions. I always include the expected result versus the actual result, the environment, like browser, OS, and build version, plus screenshots, logs, or a trace file as attachments to give developers full context. I assign a severity, based on impact, and a priority, based on urgency, and link the bug to the related test case or user story. After a fix is delivered, I retest it and move the status to verified before closing, and I keep monitoring the bug's status through the workflow to make sure nothing gets silently forgotten.

🇻🇳 Bản dịch tiếng Việt

Mỗi khi phát hiện một lỗi (defect), tôi ghi nhận nó trong một công cụ theo dõi như Jira với tiêu đề rõ ràng và các bước tái hiện lỗi chi tiết, để bất kỳ ai cũng có thể tái hiện lỗi mà không cần hỏi lại tôi. Tôi luôn nêu rõ kết quả mong đợi so với kết quả thực tế, môi trường như trình duyệt, hệ điều hành và phiên bản build, cùng với ảnh chụp màn hình, log, hoặc file trace đính kèm dưới dạng tệp đính kèm để cung cấp đầy đủ ngữ cảnh cho lập trình viên. Tôi gán mức độ nghiêm trọng dựa trên tác động, và độ ưu tiên dựa trên mức độ khẩn cấp, đồng thời liên kết bug với test case hoặc user story liên quan. Sau khi bản sửa lỗi được bàn giao, tôi kiểm thử lại và chuyển trạng thái sang đã xác nhận trước khi đóng, và tôi tiếp tục theo dõi trạng thái của bug xuyên suốt quy trình để đảm bảo không có gì bị bỏ quên âm thầm.

💻 Mẫu báo cáo bug

// Sample bug report template
Title: "Checkout button unresponsive on Safari 17 after applying coupon"

Steps to reproduce:
  1. Add any item to cart
  2. Apply a valid coupon code
  3. Click "Checkout"

Expected: User is navigated to the payment page
Actual:   Button click has no effect, no error shown

Severity:    High // blocks core flow
Priority:    P1  // fix before next release
Environment: Safari 17.2, macOS 14, build v2.3.1-rc

Attachments:
  - screenshot_checkout.png
  - console_errors.log
  - playwright-trace.zip
