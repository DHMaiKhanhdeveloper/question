QUESTION 25 · TEST AUTOMATION

What is Behavior-Driven Development (BDD), and have you used Cucumber or Gherkin?

Behavior-Driven Development (BDD) là gì, và bạn đã dùng Cucumber hay Gherkin chưa?

Sample Answer (English)

Behavior-Driven Development, or BDD, is an approach where you describe application behavior in plain, structured language using the Given, When, Then format, so QA, developers, and business stakeholders share a shared understanding of what a feature should do before any code is written. These scenarios become an executable specification, meaning the same document that describes the requirement can actually be run as an automated test. Yes, I have used Cucumber with Gherkin syntax. I write feature files with scenarios in Gherkin, and then implement step definitions in code that map each Given, When, and Then line to real automation logic, like navigating pages or clicking elements. The biggest advantage is collaboration and living documentation: anyone on the team can read a feature file and immediately understand the expected behavior, which reduces miscommunication between technical and non-technical people.

🇻🇳 Bản dịch tiếng Việt

Behavior-Driven Development, hay BDD, là một cách tiếp cận trong đó bạn mô tả hành vi (behavior) của ứng dụng bằng ngôn ngữ đơn giản, có cấu trúc theo định dạng Given, When, Then, nhờ vậy QA, developer và các bên liên quan về nghiệp vụ có được sự hiểu biết chung về việc tính năng đó nên hoạt động như thế nào trước khi viết bất kỳ dòng code nào. Những kịch bản này trở thành một đặc tả có thể thực thi, nghĩa là chính tài liệu mô tả yêu cầu cũng có thể được chạy như một bài test tự động. Đúng vậy, tôi đã dùng Cucumber với cú pháp Gherkin. Tôi viết các feature file với các scenario bằng Gherkin, sau đó cài đặt các step definition trong code để ánh xạ từng dòng Given, When, Then thành logic tự động hóa thật sự, ví dụ như điều hướng trang hoặc click vào phần tử. Lợi ích lớn nhất là sự cộng tác và tài liệu sống (living documentation): bất kỳ ai trong nhóm cũng có thể đọc một feature file và hiểu ngay hành vi mong đợi, giúp giảm hiểu lầm giữa người kỹ thuật và người không chuyên kỹ thuật.

💻 Ví dụ minh họa (Cucumber + Gherkin)

# login.feature — đặc tả có thể thực thi bằng Gherkin
Feature: User login

  Scenario: Successful login with valid credentials
    Given the user is on the login page
    When the user enters username "admin" and password "12345"
    And the user clicks the login button
    Then the user should see the dashboard page

// steps/login.steps.js — step definitions ánh xạ Gherkin sang code thật
Given('the user is on the login page', async function () {
  await this.page.goto('/login');
});

When('the user enters username {string} and password {string}',
  async function (user, pass) {
    await this.loginPage.login(user, pass);
  });

Then('the user should see the dashboard page', async function () {
  await expect(this.page).toHaveURL('/dashboard');
});
