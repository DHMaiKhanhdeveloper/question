QUESTION 4 · TEST AUTOMATION

What is the Page Object Model, and why do you use it?

Page Object Model (POM) là gì, và tại sao bạn lại dùng nó?

Sample Answer (English)

The Page Object Model, or POM, is a design pattern where I create a separate class for each page or major component of the application. Each class holds the locators and the actions for that page, so my test scripts only call high-level methods, like loginPage.login(username, password), instead of dealing with raw selectors. The main benefit is maintainability: when the UI changes, I only update the locator in one place instead of fixing it in every test. It also makes the tests much more readable, and it lets me reuse the same page methods across many test cases, which keeps the code DRY and reduces duplication.

🇻🇳 Bản dịch tiếng Việt

Page Object Model, hay POM, là một mẫu thiết kế (design pattern) trong đó tôi tạo một lớp (class) riêng cho mỗi trang hoặc mỗi thành phần chính của ứng dụng. Mỗi lớp chứa các locator và các hành động của trang đó, nhờ vậy các kịch bản test chỉ gọi những phương thức cấp cao, ví dụ loginPage.login(username, password), thay vì phải làm việc trực tiếp với các selector thô. Lợi ích chính là khả năng bảo trì: khi giao diện thay đổi, tôi chỉ cập nhật locator ở một chỗ thay vì phải sửa trong từng test. Nó cũng giúp test dễ đọc hơn nhiều, và cho phép tôi tái sử dụng cùng các phương thức của trang qua nhiều test case, giúp mã nguồn tuân theo nguyên tắc DRY và giảm trùng lặp.

💻 Ví dụ minh họa (Playwright)

// LoginPage.js — gom locator + action vào 1 class
class LoginPage {
  constructor(page) {
    this.page = page;
    this.username = page.getByTestId('username');
    this.password = page.getByTestId('password');
    this.submit   = page.getByRole('button', { name: 'Login' });
  }
  async login(user, pass) {
    await this.username.fill(user);
    await this.password.fill(pass);
    await this.submit.click();
  }
}

// Trong test — sạch & dễ đọc, không lộ selector
const loginPage = new LoginPage(page);
await loginPage.login('admin', '12345');
