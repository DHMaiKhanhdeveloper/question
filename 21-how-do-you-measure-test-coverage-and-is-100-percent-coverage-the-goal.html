QUESTION 21 · TEST AUTOMATION

How do you measure test coverage, and is 100% coverage the goal?

Bạn đo test coverage như thế nào, và 100% coverage có phải là mục tiêu không?

Sample Answer (English)

I measure test coverage using tools like Istanbul or nyc for JavaScript, which instrument the code and generate a coverage report showing line coverage, branch coverage, function coverage, and statement coverage. Line coverage tells me which lines executed, while branch coverage shows whether every if-else path was tested, which is usually a stronger signal. However, I don't treat 100% coverage as the goal. A high percentage can be misleading because a line can execute without the assertion actually verifying meaningful behavior. Instead, I use coverage as a diagnostic tool to find untested areas, and I prioritize risk-based coverage, focusing effort on critical business logic and edge cases rather than chasing every remaining percentage point, since the last few percent usually bring diminishing returns for the time invested.

🇻🇳 Bản dịch tiếng Việt

Tôi đo test coverage bằng các công cụ như Istanbul hoặc nyc cho JavaScript, những công cụ này sẽ instrument mã nguồn và tạo ra một coverage report cho thấy line coverage, branch coverage, function coverage và statement coverage. Line coverage cho biết những dòng nào đã được thực thi, còn branch coverage cho thấy liệu mọi nhánh if-else đã được test hay chưa, đây thường là một tín hiệu mạnh hơn. Tuy nhiên, tôi không xem 100% coverage là mục tiêu. Một tỷ lệ phần trăm cao có thể gây hiểu lầm vì một dòng code có thể được thực thi mà assertion lại không thực sự kiểm tra hành vi có ý nghĩa. Thay vào đó, tôi dùng coverage như một công cụ chẩn đoán để tìm ra những vùng chưa được test, và tôi ưu tiên risk-based coverage, tập trung nỗ lực vào business logic quan trọng và các edge case thay vì cố gắng đạt từng phần trăm còn lại, vì vài phần trăm cuối cùng thường mang lại lợi ích giảm dần so với công sức bỏ ra.

📊 Các loại coverage

Loại coverageÝ nghĩa
Line coverageTỷ lệ % số dòng code đã được thực thi ít nhất một lần
Branch coverageTỷ lệ % các nhánh điều kiện (if/else, switch) đã được đi qua cả hai chiều
Function coverageTỷ lệ % số hàm/method đã được gọi ít nhất một lần trong test
Statement coverageTỷ lệ % số câu lệnh (statement) đã được thực thi, tương tự line nhưng chính xác hơn với dòng nhiều lệnh
