QUESTION 23 · TEST AUTOMATION

How do you handle dynamic elements and locators that change frequently?

Bạn xử lý dynamic element và locator hay thay đổi như thế nào?

Sample Answer (English)

My first strategy is to push for stable attributes, like data-testid, that developers add specifically for automation and never change for styling reasons. When I don't have that, I prefer role-based locators, like getByRole with an accessible name, because they reflect what the user actually sees rather than the DOM structure. I try to avoid brittle XPath or CSS selectors that depend on nesting, index position, or generated class names, since those break the moment a designer refactors a component. For elements that render asynchronously, I wait for the specific element or network response to settle instead of adding fixed sleeps, which keeps the test both stable and fast. I also work closely with developers early in the sprint to get test IDs added before automation starts, so locators stay resilient as the UI evolves.

🇻🇳 Bản dịch tiếng Việt

Chiến lược đầu tiên của tôi là thúc đẩy dùng các thuộc tính ổn định, như data-testid, mà các lập trình viên thêm vào riêng cho mục đích automation và không bao giờ thay đổi vì lý do giao diện. Khi không có sẵn thuộc tính đó, tôi ưu tiên các locator dựa trên role, như getByRole với tên có thể truy cập, vì chúng phản ánh những gì người dùng thực sự nhìn thấy thay vì cấu trúc DOM. Tôi cố tránh các XPath hoặc CSS selector dễ vỡ phụ thuộc vào cấu trúc lồng nhau, vị trí chỉ số, hoặc tên class được sinh tự động, vì chúng sẽ hỏng ngay khi designer tái cấu trúc một component. Với các phần tử hiển thị bất đồng bộ, tôi chờ phần tử cụ thể hoặc phản hồi mạng ổn định thay vì thêm các lệnh sleep cố định, giúp test vừa ổn định vừa nhanh. Tôi cũng phối hợp chặt chẽ với lập trình viên ngay từ đầu sprint để thêm test ID trước khi automation bắt đầu, giúp locator luôn bền vững khi giao diện thay đổi.

💻 Ví dụ minh họa (Playwright)

// ❌ Brittle: phụ thuộc cấu trúc DOM, index, class được sinh tự động
const badBtn = page.locator('//div[3]/div[2]/span/button[1]');
const badCard = page.locator('.css-1a2b3c > div:nth-child(4)');

// ✅ Stable: data-testid do dev thêm riêng cho test
const submitBtn = page.getByTestId('checkout-submit');

// ✅ Stable: role-based, phản ánh những gì người dùng thấy
const saveBtn = page.getByRole('button', { name: 'Save changes' });

// ✅ Chờ nội dung động ổn định thay vì sleep cố định
await page.waitForResponse(resp => resp.url().includes('/api/cart') && resp.ok());
await submitBtn.waitFor({ state: 'visible' });
await submitBtn.click();
