QUESTION 14 · TEST AUTOMATION

What is data-driven testing, and how do you implement it?

Data-driven testing là gì, và bạn triển khai nó như thế nào?

Sample Answer (English)

Data-driven testing is an approach where I separate the test logic from the test data, so the same test script runs multiple times with different input values. Instead of writing a new test for every scenario, I store the data in a CSV file, a JSON file, or an array of objects, and then I loop over that dataset, feeding each record into a parameterized test function. For example, a login test can run once for valid credentials, once for an invalid password, and once for an empty username, all using the same underlying test steps. This reduces duplicate test code, makes it very easy to extend coverage by just adding a new row of data, and keeps maintenance simple, because if the flow changes I only update the logic in one place instead of in dozens of copy-pasted tests.

🇻🇳 Bản dịch tiếng Việt

Data-driven testing là cách tiếp cận trong đó tôi tách riêng logic kiểm thử khỏi dữ liệu kiểm thử, để cùng một kịch bản test có thể chạy nhiều lần với các giá trị đầu vào khác nhau. Thay vì viết một test mới cho từng kịch bản, tôi lưu dữ liệu trong file CSV, file JSON, hoặc một mảng các object, sau đó lặp qua tập dữ liệu (dataset) đó, đưa từng bản ghi vào một hàm test tham số hóa. Ví dụ, một test đăng nhập có thể chạy một lần với thông tin hợp lệ, một lần với mật khẩu sai, và một lần với tên đăng nhập rỗng, tất cả đều dùng chung các bước test bên dưới. Cách này giúp giảm mã test trùng lặp, giúp việc mở rộng phạm vi kiểm thử rất dễ dàng chỉ bằng cách thêm một dòng dữ liệu mới, và giữ việc bảo trì đơn giản, vì nếu luồng thay đổi tôi chỉ cần cập nhật logic ở một chỗ thay vì ở hàng chục test bị sao chép.

💻 Ví dụ minh họa (Playwright)

// login.data.js — dataset dùng chung cho nhiều test case
const loginCases = [
  { name: 'valid credentials',   user: 'admin',  pass: '12345',  expected: 'Dashboard' },
  { name: 'invalid password',     user: 'admin',  pass: 'wrong',  expected: 'Invalid password' },
  { name: 'empty username',       user: '',       pass: '12345',  expected: 'Username is required' },
];

// login.spec.js — 1 test logic, nhiều lần chạy với data khác nhau
for (const tc of loginCases) {
  test(`login - ${tc.name}`, async ({ page }) => {
    await page.goto('/login');
    await page.getByTestId('username').fill(tc.user);
    await page.getByTestId('password').fill(tc.pass);
    await page.getByRole('button', { name: 'Login' }).click();
    await expect(page.getByText(tc.expected)).toBeVisible();
  });
}
